# GCC Debugging Commands Cheatsheet

This cheatsheet covers essential GCC (GNU Compiler Collection) commands and flags for debugging C and C++ programs. From basic compilation to advanced debugging symbols and optimization control.

## What is GCC?

GCC is a comprehensive compiler system that supports multiple programming languages including C, C++, Objective-C, and others. For debugging purposes, GCC provides:

- Debug symbol generation (-g flags)
- Warning and error control
- Optimization level management
- Static analysis capabilities
- Address sanitizers and runtime checks

## Organization

The interactive component above organizes GCC debugging commands into key categories:

- **Basic Debugging**: Essential flags for generating debug information
- **Warning Control**: Comprehensive warning flags for catching bugs early
- **Optimization**: Managing optimization levels for debugging vs performance
- **Sanitizers**: Runtime error detection tools
- **Static Analysis**: Compile-time code analysis options
- **Advanced Debugging**: Specialized debugging features and profiling

## Key Concepts

Understanding these GCC debugging concepts will help you write more robust code:

- **Debug Symbols**: Information that helps debuggers map machine code back to source
- **Optimization Levels**: Different levels of code optimization that affect debugging
- **Sanitizers**: Runtime tools that detect memory errors, undefined behavior, etc.
- **Static Analysis**: Compile-time checks for potential issues
- **Warning Levels**: Controlling which potential issues GCC reports
