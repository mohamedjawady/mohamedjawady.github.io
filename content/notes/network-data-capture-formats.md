# Network Data Capture Formats: NetFlow, Service Logs, and PCAP Analysis

## Overview of Network Data Collection

Network traffic analysis operates on three distinct levels, each offering different trade-offs between storage requirements, analysis depth, and operational value. Understanding these formats is crucial for building effective network security monitoring capabilities.

## The Three Levels of Network Data Collection

### 1. NetFlow (Layer 3/4 Analysis)
**Characteristics:**
- Captures only network and transport layer information
- Smallest storage footprint of all collection methods
- Limited analytical value compared to other formats

**Alternative Formats:**
- **JFlow** (Juniper's implementation)
- **NetStream** (Huawei's variant)
- **sFlow** (Sampling-based flow monitoring)

**Data Elements:**
- Source and destination IP addresses
- Source and destination ports
- Protocol type (TCP, UDP, ICMP)
- Traffic volume (bytes and packets)
- Interface information
- Timing data (start/end times, duration)

**Collection Method:**
- Generated by agents on routers, firewalls, or switches
- Can be synthesized from network monitoring tools or PCAP files
- Typically transmitted via UDP port 9995
- Directly integrated with SIEM platforms

**Use Cases:**
- High-level traffic pattern analysis
- Bandwidth utilization monitoring
- Basic anomaly detection
- Network baseline establishment

### 2. Service Logs / Layer 7 Metadata
**Primary Tools:**
- **Zeek** (formerly Bro)
- **Suricata**
- Custom application parsers

**Characteristics:**
- Layer 7 application metadata only
- Moderate storage requirements (larger than NetFlow, much smaller than PCAP)
- Exceptional analytical value for security operations

**Advantages:**
- Best value for investment ratio
- Rich application-layer context
- Protocol-specific insights
- Structured log format for analysis

**Generated Log Types:**
- **conn.log** - Connection summaries
- **dns.log** - DNS queries and responses
- **http.log** - HTTP transactions
- **files.log** - File transfer activities
- **loaded_scripts.log** - Script execution tracking
- **packet_filter.log** - Filter application logs
- **stats.log** - Collection statistics

### 3. Full Packet Capture (PCAP)
**Format Variants:**
- **.pcap** - Original format
- **.pcapng** - Next Generation format with enhanced features

**PCAPNG Enhancements:**
- Higher timestamp precision
- Interface information retention
- Comment and annotation support
- Multiple capture interfaces in single file

**Characteristics:**
- Complete network traffic preservation
- Stores every transmitted byte
- Massive storage space requirements
- Contains definitive evidence for investigations

**Collection Methods:**
- Direct wire capture
- Switch mirror port (SPAN) monitoring
- May integrate with SIEM for metadata, typically analyzed separately

### PCAP Analysis Tools

**GUI-Based Analysis:**
- **Wireshark** - Interactive packet analysis
- **PacketTotal.com** - Online analysis platform
- **Moloch** - Full packet capture and analysis
- **NetworkMiner** - File and artifact extraction

**Command-Line Analysis:**
- **tcpdump** - Packet capture and filtering
- **tshark** - Wireshark's CLI interface
- **Zeek** - Metadata extraction
- **CapTipper** - Malicious traffic analysis
- **Snort/Suricata** - Intrusion detection
- **ngrep** - Network grep functionality
- **tcpxtract** - File carving from traffic

## Wireshark Filtering Techniques

### Protocol-Based Filters
```
http                    # Filter to HTTP traffic only
dns                     # Filter to DNS traffic only
http or dns            # Filter to HTTP or DNS traffic
```

### Network-Based Filters
```
ip.addr eq 1.2.3.4                    # Source or destination IP
tcp.port eq 80                        # Source or destination port
ip.addr eq 1.2.3.4 && tcp.port eq 80  # Combined IP and port filter
frame contains "search"                # Content-based filtering
```

## Strategic Implementation Considerations

### NetFlow Deployment
**Best For:**
- Network baseline establishment
- High-level anomaly detection
- Bandwidth monitoring
- Long-term trend analysis

**Integration Points:**
- Router and switch infrastructure
- SIEM platforms for correlation
- Network monitoring dashboards

### Layer 7 Metadata Collection
**Best For:**
- Security monitoring operations
- Threat hunting activities
- Incident response investigations
- Application behavior analysis

**Implementation Strategy:**
- Deploy Zeek sensors at network chokepoints
- Integrate logs with SIEM platforms
- Establish retention policies based on investigation needs

### Full Packet Capture
**Best For:**
- Deep forensic analysis
- Malware analysis and reverse engineering
- Compliance requirements
- Definitive evidence collection

**Storage Considerations:**
- Implement intelligent retention policies
- Consider selective capture based on triggers
- Plan for high-speed storage infrastructure

## Analysis Integration Framework

### SIEM Integration Strategy

**NetFlow Integration:**
- Direct ingestion via proprietary protocols
- Real-time alerting on traffic anomalies
- Historical trend analysis capabilities

**Service Log Integration:**
- Structured log ingestion
- Rich contextual information for correlation
- Enhanced detection rule capabilities

**PCAP Integration:**
- Metadata extraction for SIEM correlation
- Full packet retrieval for investigation workflows
- External tool integration for deep analysis

## Best Practices and Recommendations

### Collection Strategy
1. **Implement all three levels** for comprehensive coverage
2. **Prioritize Layer 7 metadata** for optimal value-to-cost ratio
3. **Use NetFlow for baseline** understanding and high-level monitoring
4. **Deploy selective PCAP** based on detection triggers or high-value assets

### Analysis Workflow
1. **Start with NetFlow** for initial triage and pattern identification
2. **Pivot to service logs** for detailed application-layer context
3. **Retrieve PCAP data** for definitive analysis and evidence collection

### Operational Considerations
- **Storage planning** based on traffic volumes and retention requirements
- **Processing capabilities** for real-time analysis and correlation
- **Integration architecture** for seamless analyst workflows
- **Training programs** for effective tool utilization

## Conclusion

Network data capture formats represent a spectrum of collection capabilities, each serving specific analytical needs. The most effective network security monitoring programs implement all three levels strategically:

- **NetFlow** provides the broad visibility needed for baseline understanding
- **Service logs** deliver the rich context required for effective security operations  
- **PCAP** offers the definitive evidence necessary for thorough investigations

Success lies not in choosing one format over others, but in architecting a comprehensive collection strategy that leverages the strengths of each approach while managing the associated costs and complexity.
